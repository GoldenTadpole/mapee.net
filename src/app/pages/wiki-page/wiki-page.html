1. Chunk reading

Mapee only reads chunks that are saved in single-player worlds. The mapper converts them to the latest version in its memory. The world files remain unmodified. Mapee is capable of loading and rendering worlds from Alpha to 26.1.

2. Scanning

Columns:

Mapee uses a top-down scanning technique. Each chunk comprises 256 columns, with each column representing a block on the horizontal plane. There are three types of blocks:

1) Ignored, e.g. air.
2) Transparent, e.g. water, leaves, glass.
3) Solid, e.g. stone, grass block, wood.

If the scanner hits a solid block, the column is said to be 'finished', it will not be scanned further.

Y level:

The scanner has to choose the Y level from which to start scanning the column; there are 2 configurations:

1) Set Y - Anything above the chosen Y level is ignored, which enables layered rendering for Nether and The End.
2) Heightmap - Heightmaps store the Y level of the highest block for each column.
WORLD_SURFACE - Highest non-air block.
OCEAN_FLOOR - Higest motion blocking block.
MOTION_BLOCKING - Highest motion blocking block, including water, lava, and waterlogged blocks.
MOTION_BLOCKING_NO_LEAVES - Similar, but leaves are ignored.
Block filter:

As the scanner goes through each block in the column, it has to know which of the three types apply to it. This opportunity can the employed to filter certain blocks for more interesting block formations.

3. Rendering

This stage is perhaps the most complicated one. Each column is represented by one pixel. Blocks inside columns get painted in a top-down fashion.

There are 5 steps when it comes to painting a block, say we want to get the output color:

1) Fetch the plain block color.
2) Combine biome-aware blocks with biome color. Before 1.2, Mapee defaults to minecraft:plains biome.
3) Combine sky light and block light, then darken the color based on intensity.
4) Adjust hue and brightness by altitude.
5) Darken or lighten based on neighboring block steps.
